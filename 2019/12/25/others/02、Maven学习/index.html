02、Mavan 学习

一、Maven 基本概念

Maven 是基于项目对象模型(pom),通过xml配置文件来管理项目的编译，测试，构建，报告和文档的软件项目管理工具和依赖关系的工具。

1、项目的构建

项目的构建是高度自动化，跨平台，可重用的组件。

2、依赖管理

管理依赖的JAR包，管理工程之间的依赖关系，节约空间。

3、仓库

  • 本地仓库
  • 私服仓库
  • 中央仓库

    4、POM 概念

  • groupId:组织公司的标识
  • artifactId 项目名称
  • version 项目的版本
  • packaging 项目打包的格式
  • 继承 聚合 依赖

5、Maven 生命周期

  • 验证 validate
  • 编译 compile
  • 测试 test
  • 打包 package
  • 检查 verify
  • 安装 install
  • 部署 deploy

二、Maven的命令

  • 1、清除 maven clean 删除target目录，删除的是编译文件的目录。
  • 2、编译 mvn compile 编译项目，生成target目录
  • 3、打包 mvn package 编译项目，生成jar包
  • 4、安装 mvn install 编译项目，安装jar到本地仓库

mvn archetype: create创建Maven项目
mvn compile 编译源代码
mvn deploy 发布项目
mvn test-compile 编译测试源代码
mvn test 运行应用程序中的单元测试
mvn site 生成项目相关信息的网站
mvn clean 清除项目目录中的生成结果
mvn package 根据项目生成的jar
mvn eclipse: eclipse生成eclipse项目文件
mvn jetty:run 启动jetty服务
mvn tomcat: run启动tomcat服务

-D 指定参数，如 -Dmaven.test.skip=true 跳过单元测试； -Dmaven.javadoc.skip=true 跳过javadoc
显示maven依赖树：mvn dependency:tree
显示maven依赖列表：mvn dependency:list
下载依赖包的源码：mvn dependency:sources
安装本地jar到本地仓库：mvn install:install-file -DgroupId=packageName -DartifactId=projectName -Dversion=version -Dpackaging=jar -Dfile=path

三、Maven 依赖管理

  • 依赖添加 dependencies>dependency
  • 依赖排除 exclusions>exclusion
  • 依赖传递 scope
  • 依赖冲突 防止jar冲突，最短路径。路径相同，优先覆盖。
  • 依赖的Scope 编译，运行，测试和发布
    compile 默认
    test
    runtime
    provided
    system
    import
